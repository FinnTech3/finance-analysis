The research question

Can a single framework trade S&P 500 E-mini futures profitably across multiple, shifting market regimes — without being re-tuned for each one? Most strategies live and die with the regime they were built for. The hypothesis here: instead of selecting one strategy, run eleven distinct sub-strategies concurrently and let a meta-layer dynamically weight each one based on the regime it actually fits.

Eleven sub-strategies (regime mapping)
  • Macro Regime Fusion — Fed policy, yield curve, DXY, geopolitical-risk index (all regimes)
  • ICT Smart Money Concepts — order blocks, FVGs, liquidity sweeps, OTE (trending)
  • Options Flow Sentiment — unusual activity, put/call, dark-pool prints (all regimes)
  • Trend Following (EMA Cross) — 21/55 EMA with ATR sizing (trending)
  • Statistical Arbitrage — ES/NQ & ES/RTY pairs, z-score reversion (mean-reverting)
  • VWAP Reversion — intraday session VWAP fade (range-bound)
  • Earnings Event Positioning — asymmetric setups around index-constituent earnings (event-driven)
  • Volatility Breakout — VIX-triggered ATR-expansion entries (high-vol)
  • Sector Rotation Overlay — sector relative-strength (weighting overlay)
  • Tail-Risk Hedging — OTM put spread — 3% NAV premium/month (insurance)
  • Seasonality Overlay — calendar-based bias adjustment (overlay)
Largest contributors (P&L attribution)

A short position entered before the Iran escalation in Q1 2026 produced the largest single-position gain — over +91%. A long entered at the 200-day moving average using ICT Optimal Trade Entry contributed a further +36%. Both signals were generated and executed live during market hours via the Alpaca paper API and are logged in the trade journal.

“What the test was designed to show is not that one strategy works, but that a population of strategies, intelligently weighted against the regime the market is actually in, can outperform any single one of them across the cycle.”
Architecture (one paragraph)

Signal generation runs in Python (pandas/numpy). Macro intelligence and regime labelling are delegated to the Claude API (claude-sonnet-4-6) with prompt caching enabled to reduce per-request cost. Strategy weighting uses a custom Bayesian model in R (FactoMineR); execution flows through Alpaca’s paper API. The dashboard you are looking at right now — Finn’s SQL Finance Analyser — reads the same DuckDB store the live framework writes to.
